# プレビュー方式の移行記録

## 現行仕様

MPVへ元動画と外部ASSを直接渡す「エンコード無しプレビュー」は、装飾と画面効果を再現できず、環境依存で起動しないことがあったため、標準導線から外しています。

現在のプレビュー点検画面には次の三経路があります。

1. `480p/15秒点検プレビュー`: 先頭15秒を480pでレンダーし、全体の見た目を確認する。
2. `現在字幕だけ480p`: 選択字幕の時間範囲だけを480pでレンダーする。
3. `240p/3fps軽量プレビュー`: 画質とフレームレートを下げ、装飾を含む確認動画を軽量に生成する。

いずれも `edit_plan.json` と `decoration/decoration_project.json` を入力にし、カット後タイムラインへ再マッピングした字幕を使います。

## エンコード無し方式を標準にしない理由

- MPVの外部ASSだけでは、Python生成オーバーレイやFFmpeg画面効果を再現できない。
- 元動画へ外部字幕を重ねるだけでは、カット後タイムラインの最終結果と一致しない場合がある。
- MPV本体の導入、PATH、プロセス起動が環境依存になる。
- ブラウザプレビュー、MPV、最終FFmpeg出力の三経路を維持すると不具合の切り分けが増える。

## 役割分担

- 字幕本文と基本ASSスタイルの素早い確認: ブラウザ内プレビュー
- 枠、文字連動、画面効果の局所確認: 現在字幕だけ480p
- 長めの流れを低負荷で確認: 240p/3fps軽量プレビュー
- 最終品質の確認: 480p/15秒点検または最終出力の短区間試験

## 将来MPV経路を復活させる条件

MPV経路を再導入する場合は、単なる外部字幕表示ではなく、カットEDL、通常ASS、装飾オーバーレイの対応範囲を明示し、最終出力との差分をUIへ表示する必要があります。
