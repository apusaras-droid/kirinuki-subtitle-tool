# 切り抜き動画工房 部品単位工程管理ルール

この文書は、機能を本体から切り分け、1 工程を短く保ち、異常時の修正範囲を最小化するための運用ルールを定義する。

## 1. 基本方針

- 本体に不要な処理を積まない
- 1 工程は 1 部品、1 責務、1 保存先に寄せる
- 1 回の修正で触る範囲を最小にする
- 成功した部品だけを次工程へ渡す
- 失敗したら前工程に戻さず、失敗箇所だけを再実行する

## 2. 切り分け対象

本体から切り分ける対象は、次の 4 種類に分ける。

1. 必須コア
   - 動画情報取得
   - 音声抽出
   - 文字起こし
   - 無音 / VAD 判定
   - 編集案 JSON 生成
   - 最終出力
2. 編集補助
   - 字幕の手動修正
   - カットの手動追加
   - 保護区間の手動追加
3. 装飾
   - テキスト
   - 枠
   - 文字連動エフェクト
   - 画面エフェクト
4. 付帯機能
   - プリセット管理
   - プロジェクト一覧
   - ログ閲覧
   - クリーニング

## 3. 工程の切り方

工程は「入力」「処理」「保存」「確認」の 4 段に分ける。

- 入力: ユーザー操作または CLI 引数
- 処理: core 関数のみ
- 保存: 正本 JSON / SRT / MP4 に出す
- 確認: プレビュー / ログ / 差分確認

1 工程はこの 4 段のうち、原則 1 つだけを担う。

## 4. 最小修正の原則

修正が必要になったときは、次の順で範囲を狭める。

1. まず core の関数 1 個だけを見る
2. 次に CLI の 1 コマンドだけを見る
3. 次に GUI の 1 ボタンだけを見る
4. 最後に保存 JSON の 1 キーだけを見る

同じ不具合を直すために、別画面や別コマンドへ処理を横流ししない。

## 5. 1 工程の完了条件

1 工程は、次の 3 点を満たして初めて完了とみなす。

- 成功ログが残る
- 出力ファイルが 1 つだけ確定する
- 次工程がその出力をそのまま読める

## 6. 失敗時の扱い

- 失敗した部品の中間ファイルは残す
- 成功済みの部品は消さない
- 自動 cleanup は成功時だけ実行する
- 失敗時は再実行しやすいように、ログと中間 JSON を保持する

## 7. 代表的な部品の粒度

### 音声系

- `extract-audio`
- `detect-silence`
- `detect-vad`
- `transcribe`

### 編集系

- `build-edit-plan`
- `save-subtitles`
- `manual-cut-update`
- `manual-protect-update`

### 装飾系

- `build-decoration-project`
- `render-decoration-preview`
- `build-decoration-ass`
- `export-decoration`

### 出力系

- `preview`
- `export`
- `cleanup`

## 8. ルール違反の見つけ方

次の状態は、工程が長すぎるか、部品が混ざっているサインとする。

- 1 ボタンで 3 種類以上の正本が同時に変わる
- プレビューと最終出力で別ロジックが走る
- どこで壊れたかログだけでは追えない
- 修正のたびに他画面も壊れる

この場合は工程を分解し、保存先を分け、再実行点を前に戻す。

## 9. 運用時の確認順

1. 変更対象の部品を 1 つに絞る
2. 入力と出力を 1 つずつ確認する
3. ログが 1 つの工程に紐づいているか確認する
4. 正本 JSON が 1 つだけ更新されているか確認する
5. GUI と CLI の両方で同じ core を呼んでいるか確認する
