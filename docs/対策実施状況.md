# 対策実施状況

構造定義およびリスク評価報告をもとに、以下の対策を実装した。

## 実施済み

### 1. 原子書き込み

- `edit_plan.json`
- `project.json`
- `transcript.json`
- `silences.json`
- `edit_plan_final.json`

上記は一時ファイルへ書き込み後、置換する方式に変更した。
- Windows の共有ロックに備えてリトライと失敗時の退避を入れた。

### 2. SRT出力の原子化

- `write_srt()` を一時ファイル経由に変更した。
- 字幕本文の制御文字と改行を正規化するようにした。
- 焼き込み用の字幕だけ `［］` と `/` への無害化を行うようにした。

### 3. 音声抽出の明示的なCFR指示

- `extract-audio` の `ffmpeg` 呼び出しでバージョンに応じて `-fps_mode cfr` または `-vsync cfr` を選ぶようにした。

### 4. FFmpeg配布物の厳密化

- `download-ffmpeg.bat` / `download-ffmpeg.ps1` を追加した。
- GPL版のFFmpegアセットのみを許可するようにした。
- GitHub release の digest が取得できない場合は停止する fail-safe にした。
- 既定の取得先を固定タグ `autobuild-2026-06-15-15-03` に寄せた。

### 5. セットアップ導線

- `setup.bat` を追加した。
- `download-runtime.bat` と `download-ffmpeg.bat` を順番に呼び、最後に `launch.bat` を起動する。
- 既に導入済みなら Python依存と FFmpeg の再取得をスキップするようにした。
- `setup.bat` 実行時に、実際に何がスキップされたかを `logs/setup-status.txt` へ出すようにした。

### 6. 編集データの世代管理

- `edit_plan.json` と `transcript.json` の更新時に履歴を残すようにした。
- `.edit_plan.json.history` のような履歴フォルダへ過去版を保存する。
- 履歴は上限件数でローテーションする。

### 7. 中間ファイル整理

- `cleanup` CLI を追加した。
- `audio/source_range.wav`、`preview/`、`analysis/`、`temp/segments/`、生字幕群を整理できる。
- `run-pipeline --auto-cleanup` でパイプライン後に自動整理できる。
- `run-pipeline --auto-cleanup` は成功時のみ動き、失敗時は中間ファイルを保持する。

## 追加で検討する項目

- `edit_plan.json` の世代管理
- 長尺処理時の中間ファイル整理
- FFmpeg の配布元やバージョンの完全固定
- SRTや字幕本文の厳密なエスケープ強化
