# 切り抜き動画工房 相関関係メモ

このメモは、実装や運用の文脈が抜けたときに「何と何がつながっているか」をすぐ確認するためのものです。

## 1. まず見るもの

- 仕様全体: `docs/アーキテクチャ再設計案.md`
- 工程粒度: `docs/部品単位工程管理ルール.md`
- 修正実装順: `docs/修正実装フローチャート.md`
- 実行順: `docs/実装フローチャート.md`
- 外部利用: `docs/外部CLI連携.md`
- AI 引き継ぎ: `docs/AI向け引き継ぎ説明書.md`
- 画面別一覧: `docs/画面別ボタン索引.md`
- ボタン対応表: `docs/ボタンCLI索引.md`
- 部品別一覧: `docs/部品別工程表.md`

## 2. 正本の関係

### `project.json`

プロジェクトの基本情報。

- プロジェクト名
- 元動画情報
- UI 状態
- 既定の出力設定
- decoration 参照

### `edit_plan.json`

字幕とカットの正本。

- 残す区間
- 削除区間
- 字幕時刻
- 話者ラベル
- シーン情報
- 手動修正

### `decoration/decoration_project.json`

装飾の正本。

- 字幕スタイル
- 枠プリセット
- 文字連動エフェクト
- 画面エフェクト
- シーンごとの割り当て

### `presets/*.json`

共通初期値。

- `emotion_presets`
- `subtitle_style_presets`
- `frame_presets`
- `effect_groups`
- `screen_effect_stacks`
- `layout_presets`

## 3. GUI と CLI の関係

### GUI

GUI は編集と確認のための窓口です。

- プロジェクトを開く
- 設定を変える
- プレビューを見る
- 最終出力を押す

GUI は計算の本体を持たない。

### CLI

CLI は処理単位の実行器です。

- `probe`
- `extract-audio`
- `transcribe`
- `detect-silence`
- `detect-vad`
- `build-edit-plan`
- `render-preview`
- `export-final`
- `decorate-preview`
- `decorate-export`

CLI は GUI の別実装ではない。

## 4. 処理の相関

### 音声系

`input.mp4` -> `source_range.wav` -> `transcript.json`

- Whisper は本文生成
- VAD / 無音検出は区間生成
- 解析音声がある場合はそれを補助入力に使う

### 編集系

`transcript.json` + `silences` + `manual cuts` -> `edit_plan.json`

- ここで時間変換を確定する
- 以後の出力は `edit_plan.json` を参照する

### 装飾系

`edit_plan.json` + `decoration_project.json` -> 装飾プレビュー / 装飾最終出力

- 字幕本文と枠とエフェクトをここで反映する
- 画面エフェクトは export で再現可能なものを優先する

## 5. 時刻の相関

### 元動画時間

元動画そのままの時刻。

### 指定範囲相対時間

処理対象区間の先頭を 0 秒にした時刻。

### 完成動画時間

カット後の動画時刻。

## 6. エラー時の見方

1. `logs/server-<port>.log`
2. `logs/app_audit.jsonl`
3. `projects/<project_id>/temp/logs/processing.jsonl`
4. `projects/<project_id>/temp/logs/whisper_cpp.log`
5. `projects/<project_id>/temp/logs/final_burn.log`

## 7. 変更時の確認順

何かを変えるときは、次の順で影響を確認する。

1. core の関数
2. CLI の引数と JSON
3. GUI のボタンや選択肢
4. 保存される JSON
5. 最終出力の反映

## 8. 工程を短くする考え方

- 1 変更は 1 保存先まで
- 1 不具合は 1 部品まで
- 画面をまたぐ修正は、core が同じか先に確認する
- 本体に不要なものは decoration / preset / optional に退避する
- 迷ったら `docs/部品単位工程管理ルール.md` を先に見る

## 9. 迷ったときの判断

- 字幕とカットの問題なら `edit_plan.json`
- 見た目の問題なら `decoration_project.json`
- 共通既定値の問題なら `presets/*.json`
- 起動や再実行の問題なら CLI と BAT
- 実行差分の問題なら core と GUI の境界

## 10. ボタンとCLIの対応

### プロジェクト系

- `動画を選択` + `読み込み` -> `new-project`
- `プロジェクト一覧` -> `run-pipeline` の管理対象を選ぶ画面
- `プロジェクト保存` -> `project.json` / `edit_plan.json` / `decoration_project.json` の保存
- `上書き保存` -> 同じプロジェクトIDに対する再保存
- `削除` -> CLI では `cleanup` に近いが、プロジェクト削除は GUI 側の管理機能

### 音声・字幕系

- `動画情報取得` -> `probe`
- `音声を抽出` -> `extract-audio`
- `文字起こしを実行` -> `transcribe`
- `無音区間を検出` -> `detect-silence`
- `VADを使う` / `声抽出を使う` -> `transcribe` / `detect-vad` のオプション
- `カット案を作成` -> `create-edit-plan`
- `字幕を保存` -> `save-subtitles`
- `字幕のscene_idを再割当` -> 直結 CLI は薄い。`edit_plan.json` 更新の GUI 操作に近い

### プレビュー・出力系

- `仮出力を作成` -> `preview`
- `手動カットを仮出力` -> `preview/manual-cuts` 相当。CLI では `edit_plan.json` を先に反映して `preview`
- `最終出力` -> `export`
- `字幕を焼き込む` -> `export --burn-subtitles`
- `出力プリセット` -> `export --output-profile`

### デコレーション系

- `デコレーション画面` -> GUI 専用の編集ビュー
- `字幕から生成` -> `load_project_decoration` の再構築
- `保存` -> `/api/projects/decoration`
- `ASS出力` -> `build_decoration_ass` / `/api/decoration/ass`
- `プレビュー点検` -> GUI 専用の確認ビュー
- `480pプレビュー作成` -> `render_decoration_video` / `/api/decoration/render`
- `共通へ保存` / `共通プリセットへ保存` -> `save_shared_decoration_presets` -> `data/shared/decoration_presets.json`

### 波形・手動編集系

- `開始位置に設定` / `終了位置に設定` -> GUI での範囲入力補助
- `波形編集` の `カット開始` / `カット終了` -> `edit_plan.json` の `manual_cut_segments` 更新
- `保護区間` 系 -> `edit_plan.json` の `protected_segments` 更新
- `選択解除` / `最後を削除` -> GUI の編集状態を戻す操作

### ルール

- GUI でしかできないものは「表示・編集の便宜」
- CLI でしかできないものは「一括処理・自動化」
- 両方あるものは core の同じ関数を通す
- 片方にしかないものは、将来的に core へ寄せる候補

## 10. 実ボタン名の対応補足

### 編集画面

- `動画を選択` -> ファイル選択のみ。実行自体は `読み込み`
- `読み込み` -> `new-project`
- `動画情報取得` -> `probe`
- `音声を抽出` -> `extract-audio`
- `文字起こしを実行` -> `transcribe`
- `無音区間を検出` -> `detect-silence`
- `カット案を作成` -> `create-edit-plan`
- `字幕を保存` -> `save-subtitles`
- `仮出力を作成` -> `preview`
- `手動カットを仮出力` -> 先に `edit_plan.json` を反映してから `preview`
- `最終出力` -> `export`

### 範囲・波形編集

- `開始位置に設定` -> 現在再生位置を開始欄へ反映
- `終了位置に設定` -> 現在再生位置を終了欄へ反映
- `現在の開始/終了を区間に反映` -> `source_range` を更新
- `10分で分割` / `15分で分割` / `20分で分割` / `30分で分割` -> 区間分割の補助
- `カット開始` / `カット終了` -> `manual_cut_segments` を追加
- `選択解除` -> 現在の手動選択を破棄
- `最後を削除` -> 最新の手動カットを削除

### プロジェクト一覧

- `プロジェクト一覧` -> 保存済みプロジェクトの再開
- `再読込` -> `GET /api/projects`
- `開く` -> 対象プロジェクトを再ロード
- `削除` -> プロジェクトディレクトリ削除

### 設定画面

- `VADを使う` -> `transcribe` / `detect-vad` の制御
- `声抽出を使う` -> 解析用音声の前段処理
- `字幕フォント` / `字幕サイズ` / `字幕の黒縁` -> `export --burn-subtitles` の見た目
- `出力プリセット` -> `export --output-profile`

### デコレーション画面

- `字幕から生成` -> `decoration_project.json` を字幕から再構築
- `現在字幕からセット作成` -> 選択中字幕から effect group を複製
- `ASS出力` -> `build_decoration_ass`
- `保存` -> `save_project_decoration`
- `JSON出力` -> decoration のエクスポート
- `480pプレビュー作成` -> `render_decoration_video`
- `再生` / `停止` -> 装飾プレビュー動画の再生制御

### 判定の優先順位

1. まず画面のボタン名を見る
2. 次に API エンドポイントを見る
3. 最後に CLI コマンドを見る
4. GUI 専用なら CLI 名は書かない
5. CLI 専用なら GUI ボタン名は書かない
