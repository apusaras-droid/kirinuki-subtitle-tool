# Security Policy

## Supported versions

正式なGitHub Releaseとして公開された最新バージョンを優先して修正します。開発途中のコミットや古いReleaseについて、同じ期間の保守を保証するものではありません。

## Reporting a vulnerability

脆弱性、任意ファイルアクセス、コマンド注入、秘密情報漏えいなどを発見した場合は、公開Issueへ詳細を直接投稿しないでください。

GitHubリポジトリのPrivate vulnerability reportingが有効な場合は、その経路を使用してください。有効でない場合は、再現手順からAPIキー、個人パス、入力動画、字幕本文などの機密情報を除き、まず最小限の概要だけをリポジトリ所有者へ連絡してください。

報告に含める情報:

- 対象バージョンまたはコミットSHA
- OS、Python、FFmpeg、whisper.cppのバージョン
- 影響範囲
- 最小再現手順
- 期待結果と実際の結果
- 秘密情報を除去したログ

## Local-only boundary

本アプリは `127.0.0.1` でのローカル利用を前提とします。LANやインターネットへバインドして公開するための認証・認可機構は提供していません。

Gemini APIは任意機能です。APIキーは `data/private/` または環境変数で管理し、Gitへコミットしないでください。動画・音声・字幕をGeminiへ送信する場合は、利用者自身が送信内容とGoogleの利用条件を確認してください。

## Response

報告を確認後、影響評価、回避策、修正版の予定を可能な範囲で連絡します。秘密情報が漏えいした場合は、履歴修正より先に該当キーを失効・再発行してください。
